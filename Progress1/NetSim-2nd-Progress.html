📚 Master Hub: Core AWS Networking Deep Dive

Panduan teknis mendalam dan komprehensif standardisasi arsitektur cloud AWS regional hingga global tepi.

☁️ 1. Amazon VPC (Virtual Private Cloud) Fondasi Regional

Amazon VPC memisahkan infrastruktur Anda secara logis di dalam satu AWS Region tertentu secara terisolasi. VPC tidak bisa lintas Region!

A. Komponen Pengalamatan & Pembagian Jaringan

• VPC CIDR Block: Penentuan rentang alamat IP menggunakan notasi Classless Inter-Domain Routing. Contoh: 10.0.0.0/16 menghasilkan total 65.536 IP privat.

• Subnet: Rentang IP terisolasi yang terikat wajib pada 1 Availability Zone (AZ).

- Public Subnet: Memiliki rute langsung ke Internet Gateway. Bagus untuk Web Server / Load Balancer.

- Private Subnet: Terisolasi dari internet luar. Tempat aman untuk Database RDS / backend sensitif.

⚠️ Aturan AWS: Di setiap subnet, AWS memesan 5 alamat IP pertama dan terakhir untuk internal routing, DNS, dan broadcast!

B. Komponen Konektivitas (Gateways & Routing)

• Internet Gateway (IGW): Penghubung VPC ke internet luar secara dua arah (Inbound & Outbound) bersifat highly available.

• NAT Gateway: Diletakkan wajib di Public Subnet agar resource di Private Subnet bisa download patch/update ke internet, tetapi memblokir koneksi liar dari internet luar masuk ke dalam.

• Route Table (Tabel Rute): Penentu arah paket data. Rute lokal bawaan (10.0.0.0/16 -> local) menjamin seluruh subnet saling kenal. Rute default internet dinyatakan via 0.0.0.0/0.

C. Lapisan Keamanan Jaringan (Firewall Berlapis)

Fitur Security Group (SG) Network Access Control List (NACL)
Tingkat Operasi Tingkat Instance (Virtual Interface / ENI) Tingkat Subnet
Sifat Status Stateful Stateless
Jenis Aturan Hanya mendukung aturan mengizinkan (Allow). Deny by default. Mendukung aturan Allow dan Deny secara eksplisit.
Evaluasi Aturan Seluruh aturan dicek bersamaan tanpa urutan. Dievaluasi berurutan berdasarkan nomor terkecil (Aturan 100 sebelum 200).

🗺️ 2. AWS Route 53 (Deep Dive)

Amazon Route 53 memanfaatkan arsitektur Anycast global agar kueri resolusi domain dari user selalu mendarat di DNS server terdekat dengan latensi minimal.

A. Terminologi dan Struktur Data DNS AWS

  • Hosted Zone: Wadah records DNS domain Anda. Tersedia versi Public untuk rute luar internet dan Private khusus resolusi internal VPC.

DNS Record Types Utama:

• A Record: Memetakan nama domain ke alamat IPv4 fisik.

• CNAME Record: Memetakan domain ke nama domain lain. Tidak bisa ditaruh di Zone Apex (root domain).

• Alias Record: Fitur eksklusif AWS. Bisa dipakai di Zone Apex untuk memetakan nama root domain langsung ke AWS resources (CloudFront/Load Balancer) tanpa overhead biaya DNS.

B. Bedah Mendalam 7 Kebijakan Routing

1. Simple Routing:

Margarahkan domain ke satu atau sekumpulan IP statis secara round-robin tanpa filter kesehatan.

2. Weighted Routing:

Membagi persentase trafik berdasarkan bobot angka (cocok untuk uji coba Blue-Green Deployment).

3. Latency-based Routing:

Megarahkan user otomatis ke AWS Region yang memberikan waktu tunda paling kecil.

4. Failover Routing:

Mengalihkan rute 100% ke disaster recovery server (Passive) jika server utama (Active) terdeteksi down oleh Health Check.

5. Geolocation Routing:

Membasi atau mengarahkan alur data berdasarkan benua/negara asal IP pengguna.

6. Geoproximity Routing:

Menghitung jarak koordinat absolut dan memanipulasinya lewat nilai 'Bias'.

7. Multi-value Answer:

Mengbalikkan hingga 8 record IP yang dipastikan sehat walafiat lewat integrasi kueri ter-monitoring.

⚡ 3. Amazon CloudFront (Deep Dive)

Lapisan Content Delivery Network (CDN) global yang memotong jarak transmisi data statis maupun dinamis aplikasi Anda.

A. Komponen Edge

  • Edge Locations: Titik server mini luar Region penampung cache terdekat dari end-user.
  • Regional Edge Caches (REC): Lapisan tengah penyarian badai kueri agar tidak menghantam langsung server origin pusat.

B. Aturan Enkripsi Jaringan

Mengamankan S3 Bucket via Origin Access Control (OAC) sehingga data di S3 tidak bisa dicuri secara bypass langsung melainkan wajib melewati filter otentikasi CloudFront.

🤝 4. Sinergi Arsitektur Utuh (Full Systems Integration)

Kolaborasi orkestrasi global Route 53 + CloudFront di tepi perimeter digabungkan dengan ketatnya barikade Amazon VPC regional.

Alur Sekuensial Paket Data:

1. User melempar domain $\rightarrow$ **Route 53** menyuplai IP Edge **CloudFront** terdekat via Alias Record.

2. CloudFront memproses jika terjadi *Cache Hit* langsung dipulangkan ke user. Jika *Cache Miss*, dilempar ke internal **VPC**.

3. Application Load Balancer di Public Subnet menerima data, memverifikasi token security group lalu memproses ke EC2 di Private Subnet dengan super aman.
